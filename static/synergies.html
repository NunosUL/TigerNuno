Pipeline Synergies

The ingestion pipeline and the RAG chat are two separate processes — but every decision made in ingestion was built to make chat work correctly. Here's how they connect.

⚙ Ingestion ⬡ Shared Contract 💬 Chat / RAG

The 7 synergies in detail

⚡
① Same Embedding Model

Both sides use text-embedding-3-large. This is the most critical constraint — vectors only make sense for similarity search if they were created in the same 3072-dimensional space. Different models = meaningless results.

text-embedding-3-large 3072 dims critical constraint
⬡
② Shared Index — the Handoff Point

Azure AI Search's wiki-index is the single point where ingestion ends and chat begins. Ingestion writes; chat reads. The index schema was designed to serve both — upsert by ID for ingestion, hybrid search fields for chat.

wiki-index upsert hybrid search
✂️
③ Chunk Size Shapes Answer Quality

The 1000-char / 150-overlap split was chosen so 8–10 chunks fit in GPT-4o's 12,000-char context budget while staying semantically meaningful. Too large = fewer sources. Too small = loss of context at boundaries.

1000 chars 150 overlap 12k budget
🔑
④ Deterministic IDs Enable Clean Upserts

Chunk IDs are built from the page path + chunk index. This means re-running ingestion overwrites exactly the right vectors — never creating duplicates. The chat always queries a clean, current index with no stale data.

path-based ID no duplicates always current
📄
⑤ Metadata Becomes Citations

path and url are stored at ingestion alongside every chunk. At query time, these are retrieved with the chunks and surfaced in the chat UI as clickable source citations — providing traceability back to the original wiki page.

path url source citations
🔒
⑥ Change Detection Keeps Index Fresh

The MD5 hash manifest means only changed pages get re-embedded. The chat always gets fresh content after a wiki update, without a costly full re-index. If ingestion fails mid-run, the manifest isn't saved — so the next run retries safely.

MD5 per page skip unchanged safe retries
🔍
⑦ Text Field Enables Hybrid Search

The text field stored at ingestion is marked searchable in the index schema, enabling Azure AI Search's BM25 full-text engine alongside vector search. The LangChain splitter preserves natural sentences, making keyword matching effective.

BM25 keywords vector similarity RRF fusion

The key insight

The ingestion pipeline and the RAG chat are decoupled in time — ingestion runs on demand when the wiki changes, chat runs on every user question — but they are tightly coupled in contract: same embedding model, same index schema, same metadata fields. Change any one of those and both sides break. This is why the two pipelines live in the same repository and share the same .env configuration.
