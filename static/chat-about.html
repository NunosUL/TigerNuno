How Chat Works

Every question goes through a 5-step RAG pipeline — your query is embedded, matched against the wiki index, and answered by GPT-4o using only the retrieved context.

RAG Query Pipeline — end to end

Pipeline steps explained

⚡

① Embed Query

Your question is converted into a 3072-dimensional vector using the same text-embedding-3-large model that was used during ingestion. This ensures the question vector lives in the same semantic space as the stored chunk vectors, making similarity meaningful.

Azure OpenAI text-embedding-3-large 3072 dims
🔍

② Hybrid Search

Azure AI Search runs two searches simultaneously: a vector similarity search using the query embedding (finds semantically related chunks even with different wording), and a full-text keyword search on the raw question text (catches exact term matches). Results are merged using Reciprocal Rank Fusion (RRF), which rewards chunks that rank well in both searches. The top-K chunks are returned.

Azure AI Search Vector similarity Full-text BM25 RRF fusion
📋

③ Context Builder

The retrieved chunks are ranked by relevance score, deduplicated by ID, and soft-capped per wiki page so that no single page dominates the context window. Chunks are concatenated with their source path as a label until the 12,000-character budget is reached. The unique source pages are recorded for citation.

Deduplication Token budget Source metadata
🧠

④ LLM Generation

The context and question are sent to GPT-4o with a strict system prompt: "Answer using ONLY the provided context. Cite sources. If the context is insufficient, say so." Temperature is set to 0.2 for factual, deterministic answers. The model cannot hallucinate facts not present in the retrieved wiki content.

Azure OpenAI GPT-4o temp 0.2 max 1024 tokens
✦

⑤ Format Response

The raw answer is returned alongside the list of source wiki pages as clickable citations, and a confidence signal computed from the number of chunks retrieved, their relevance scores, and whether the model indicated uncertainty in its answer.

Answer text Source citations Confidence signal

Confidence Signal

High 4+ chunks retrieved with strong relevance scores and no uncertainty language detected in the answer.
Medium 2+ chunks retrieved with moderate scores. Answer is likely correct but context coverage is partial.
Low Few or weakly-matched chunks, or the model explicitly stated it could not find enough information.
