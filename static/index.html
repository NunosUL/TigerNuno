⚠ Maintenance

Clear change-detection manifest rag_hashes.json
The manifest stores a hash of every indexed record so unchanged content is skipped. Delete it when you want to force a full re-index — for example after updating ingestion code, or when the index seems stale despite a successful crawl. The next run will re-chunk, re-embed and re-upload every record.
Delete crawl snapshot rag_ulpsi_NetProjects10.jsonl
The snapshot is the raw JSONL file produced by crawling — it contains every wiki page, source code file, test case, work item, and commit diff fetched from Azure DevOps. Delete it only if you suspect the snapshot itself is corrupted or you want to free up storage. The next run with Crawl ON will rebuild it automatically. Running with Crawl OFF after deleting will fail (nothing to load).
